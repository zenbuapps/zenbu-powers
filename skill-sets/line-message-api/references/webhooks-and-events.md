# Webhooks

When an event occurs (user adds the LINE Official Account, sends a message, etc.),
the LINE Platform sends an HTTPS POST request to the channel's webhook URL.
Make event processing asynchronous. The LINE Platform IP address is not disclosed —
use signature validation, not IP allowlists.

## Request headers

| Header | Use |
|---|---|
| `x-line-signature` | Used for signature validation. Header field names are case-insensitive — handle without case distinction. |

## Request body

| Property | Type | Description |
|---|---|---|
| `destination` | String | User ID of the bot that should receive the events. Matches `U[0-9a-f]{32}`. |
| `events` | Array | Array of webhook event objects. May be an empty array for a communication check. |

## Response

The bot server **must return HTTP `200`** after receiving the POST request.
A POST with an empty `events` array (`{"destination":"...","events":[]}`) is a
communication check — still return `200`.

## Signature validation

Verify `x-line-signature` before processing events. The signature is the Base64-encoded
HMAC-SHA256 digest of the **raw request body string** keyed by the **channel secret**.
Compare it against the `x-line-signature` header.

Node.js:
```javascript
const crypto = require("crypto");
const channelSecret = "..."; // Channel secret string
const body = "...";          // Raw request body string
const signature = crypto.createHmac("SHA256", channelSecret).update(body).digest("base64");
// Compare `signature` with the x-line-signature request header
```

Python:
```python
import base64, hashlib, hmac
channel_secret = '...'  # Channel secret string
body = '...'            # Raw request body string
hash = hmac.new(channel_secret.encode('utf-8'), body.encode('utf-8'), hashlib.sha256).digest()
signature = base64.b64encode(hash)
# Compare `signature` with the x-line-signature request header
```

PHP:
```php
$hash = hash_hmac('sha256', $httpRequestBody, $channelSecret, true);
$signature = base64_encode($hash);
```

Go: HMAC-SHA256 the body with `[]byte("<channel secret>")`, then compare with
`base64.StdEncoding.DecodeString(req.Header.Get("x-line-signature"))` using `hmac.Equal`.

# Webhook settings endpoints

## Set webhook endpoint URL

`PUT https://api.line.me/v2/bot/channel/webhook/endpoint`

Sets the channel webhook endpoint URL. May take up to 1 minute to apply (caching).
URL rules: valid HTTPS URL, ≤ 500 characters. Rate limit: 1,000/min.

Headers: `Authorization: Bearer {token}`, `Content-Type: application/json`
Body: `endpoint` (String, required) — a valid webhook URL.
Response: `200` + `{}`. Error: `400` (invalid URL).

```sh
curl -X PUT -H 'Authorization: Bearer {token}' -H 'Content-Type:application/json' \
-d '{"endpoint":"https://example.com/hoge"}' \
https://api.line.me/v2/bot/channel/webhook/endpoint
```

## Get webhook endpoint information

`GET https://api.line.me/v2/bot/channel/webhook/endpoint`

Headers: `Authorization: Bearer {token}`, `Content-Type: application/json`
Response 200: `endpoint` (String, webhook URL), `active` (Boolean — webhook usage enabled).
Error: `404` (webhook URL not set).

```json
{ "endpoint": "https://example.com/test", "active": true }
```

## Test webhook endpoint

`POST https://api.line.me/v2/bot/channel/webhook/test`

Checks if the configured webhook endpoint can receive a test event. Rate limit: 60/hour.

Headers: `Authorization: Bearer {token}`, `Content-Type: application/json`
Body: `endpoint` (String, optional). **With** `endpoint`: validates that URL and sends a
test event to it. **Without** `endpoint` (`{}`): sends a test event to the channel's
configured URL (`404` if none configured).

Response 200:

| Property | Type | Description |
|---|---|---|
| `success` | Boolean | Communication result (`true`/`false`) |
| `timestamp` | String | ISO 8601 timestamp |
| `statusCode` | Number | HTTP status code; 0/negative if no response received |
| `reason` | String | `OK`, `COULD_NOT_CONNECT`, `REQUEST_TIMEOUT`, `ERROR_STATUS_CODE`, `UNCLASSIFIED` |
| `detail` | String | Detail of the response |

```json
{ "success": true, "timestamp": "2020-09-30T05:38:20.031Z", "statusCode": 200, "reason": "OK", "detail": "200" }
```

Errors: `400` (invalid URL), `404` (webhook URL not set).

# Webhook event objects

JSON objects containing events generated by the LINE Platform. Generated event objects
omit properties without a value. A single webhook may contain multiple event objects
from different users.

```json
{
  "destination": "xxxxxxxxxx",
  "events": [
    { "type": "message", "message": {"type":"text","id":"14353798921116","text":"Hello, world"},
      "timestamp": 1625665242211, "source": {"type":"user","userId":"U80696558e1aa831..."},
      "replyToken": "757913772c4646b784d4b7ce46d12671", "mode": "active",
      "webhookEventId": "01FZ74A0TDDPYRVKNK77XKC3ZR", "deliveryContext": {"isRedelivery": false} }
  ]
}
```

## Common properties (all event objects)

| Property | Type | Description |
|---|---|---|
| `type` | String | Event type identifier |
| `mode` | String | Channel state. `active`: can send reply/push. `standby`: waiting — no reply token, bot must not send messages. |
| `timestamp` | Number | UNIX time of the event in milliseconds (time of event, not redelivery) |
| `source` | Object | (Not always) Source user / group / room object. Omitted in failed account link events. |
| `webhookEventId` | String | ULID-format ID uniquely identifying the event |
| `deliveryContext.isRedelivery` | Boolean | `true` if this is a redelivered webhook event |

## Source objects

**Source user**: `type` = `user`, `userId`.
```json
{ "type": "user", "userId": "U4af4980629..." }
```

**Source group chat**: `type` = `group`, `groupId`, `userId` (not always — only in
message events, only for LINE iOS/Android users).
```json
{ "type": "group", "groupId": "Ca56f94637c...", "userId": "U4af4980629..." }
```

**Source multi-person chat (room)**: `type` = `room`, `roomId`, `userId` (not always —
same conditions as group).
```json
{ "type": "room", "roomId": "Ra8dbf4673c...", "userId": "U4af4980629..." }
```

## Message event

`type` = `message`. Contains a sent message; the `message` property holds a message
object of the corresponding type. Can be replied to.

| Property | Type | Description |
|---|---|---|
| `replyToken` | String | Reply token for Send reply message |
| `message` | Object | Message content object — type: text, image, video, audio, file, location, sticker |

### message: Text

| Property | Type | Description |
|---|---|---|
| `id` | String | Message ID |
| `type` | String | `text` |
| `quoteToken` | String | Quote token of this message |
| `markAsReadToken` | String | Read token (no expiration) for Mark messages as read |
| `text` | String | Message text. LINE emojis appear as `(name)`; mentions appear as `@name`. |
| `emojis` | Array | (Not always) LINE emoji objects; present when `text` contains LINE emojis |
| `emojis[].index` | Number | Index of the emoji char in `text` (0-based) |
| `emojis[].length` | Number | Length of the LINE emoji string |
| `emojis[].productId` | String | LINE emoji set product ID |
| `emojis[].emojiId` | String | LINE emoji ID within the set |
| `mention` | Object | (Not always) Present when `text` contains a mention |
| `mention.mentionees[]` | Array | Mention objects, max 20 |
| `mention.mentionees[].index` | Number | Index of the mention in `text` (0-based) |
| `mention.mentionees[].length` | Number | Length of the mention text |
| `mention.mentionees[].type` | String | `user` or `all` (entire group) |
| `mention.mentionees[].userId` | String | (Not always) User ID — only when type is `user` and the user consented |
| `mention.mentionees[].isSelf` | Boolean | (Not always) `true` if the mention is to the receiving bot |
| `quotedMessageId` | String | (Not always) Message ID of a quoted message |

### message: Image

`id`, `type`=`image`, `quoteToken`, `markAsReadToken`,
`contentProvider.type` (`line` = retrievable via Get content, or `external`),
`contentProvider.originalContentUrl` / `previewImageUrl` (only when `external`),
`imageSet.id` / `imageSet.index` / `imageSet.total` (only when multiple images sent at once;
`index`/`total` absent on LINE Android ≤11.15). Webhook delivery order for image sets is undefined.

### message: Video

`id`, `type`=`video`, `quoteToken`, `markAsReadToken`, `duration` (Number ms, not always),
`contentProvider.type` (`line`/`external`), `contentProvider.originalContentUrl` / `previewImageUrl` (when `external`).

### message: Audio

`id`, `type`=`audio`, `markAsReadToken`, `duration` (Number ms, not always),
`contentProvider.type` (`line`/`external`), `contentProvider.originalContentUrl` (when `external`).

### message: File

`id`, `type`=`file`, `markAsReadToken`, `fileName` (String), `fileSize` (Number bytes).
Binary retrievable via Get content.

### message: Location

`id`, `type`=`location`, `markAsReadToken`, `title` (not always), `address` (not always),
`latitude` (Decimal), `longitude` (Decimal).

### message: Sticker

`id`, `type`=`sticker`, `quoteToken`, `markAsReadToken`, `packageId`, `stickerId`,
`stickerResourceType` (one of `STATIC`, `ANIMATION`, `SOUND`, `ANIMATION_SOUND`, `POPUP`,
`POPUP_SOUND`, `CUSTOM`, `MESSAGE`, `NAME_TEXT` (discontinued), `PER_STICKER_TEXT`
(discontinued) — handle unknown future types), `keywords` (Array of up to 15 strings,
not always), `text` (not always — only message stickers, max 100 chars), `quotedMessageId`
(not always). Sticker image itself can't be retrieved.

## Unsend event

`type` = `unsend`. `unsend.messageId` (String) — message ID of the unsent message.
Respect the user's intent: make the message no longer visible/usable.

## Follow event

`type` = `follow`. Account added as friend or unblocked. Can be replied to.
`replyToken`, `follow.isUnblocked` (Boolean — `true`: unblocked, `false`: added as friend;
accuracy not guaranteed).

## Unfollow event

`type` = `unfollow`. Account was blocked. No reply token.

## Join event

`type` = `join`. Account joined a group/multi-person chat. Can be replied to. `replyToken`.

## Leave event

`type` = `leave`. Account removed from / left a group/multi-person chat. No reply token.

## Member join event

`type` = `memberJoined`. A user joined a group/room the account is in. Can be replied to.
`replyToken`, `joined.members` (Array of source user objects).

## Member leave event

`type` = `memberLeft`. A user left a group/room the account is in. No reply token.
`left.members` (Array of source user objects).

## Postback event

`type` = `postback`. A postback action was performed. Can be replied to.
`replyToken`, `postback.data` (String — postback data), `postback.params` (Object).

`postback.params` for **date-time selection action** (RFC3339):

| Property | Format | Description |
|---|---|---|
| `date` | full-date | Date selected (only in `date` mode) |
| `time` | time-hour ":" time-minute | Time selected (only in `time` mode) |
| `datetime` | full-date "T" time-hour ":" time-minute | Datetime selected (only in `datetime` mode) |

`postback.params` for **rich menu switch action**:

| Property | Description |
|---|---|
| `newRichMenuAliasId` | (Not always) Rich menu alias ID switched to; absent if switch failed |
| `status` | `SUCCESS`, `RICHMENU_ALIAS_ID_NOTFOUND`, `RICHMENU_NOTFOUND`, `FAILED` |

## Video viewing complete event

`type` = `videoPlayComplete`. User finished a video with the specified `trackingId`.
`replyToken`, `videoPlayComplete.trackingId` (String — same value as the video message's
`trackingId`). Not a view count. Not supported for video in imagemap/Flex messages.

## Beacon event

`type` = `beacon`. User entered a LINE Beacon range. Can be replied to.
`replyToken`, `beacon.hwid` (String — hardware ID), `beacon.type` (`enter`, `banner`, `stay`),
`beacon.dm` (String — device message, not always; `stay` repeats at ≥10s intervals).

## Account link event

`type` = `accountLink`. User linked their LINE account with a provider's service.
`replyToken` (not always — absent if linking failed), `link.result` (`ok` / `failed`),
`link.nonce` (String — the nonce specified during verification). `source` is absent if linking failed.

## Membership event

`type` = `membership`. User joined/renewed/left a membership of the account.
`replyToken`, `membership.type` (`joined`, `left`, `renewed`), `membership.membershipId` (Number).
No event if the user hasn't consented to profile access.
